1. What is the Hypertext Transfer Protocol (HTTP)?

Hypertext Transfer Protocol is an application layer protocol which is used to transmit hypermedia documents such as HTML(Hyper Text Markup Language). It was designed with the purpose of communication between web browsers and web servers along with various other purposes. It tends to follow a classical client-server model. whereby a client sends in a request and waits for a response from the server.

2. Why is HTTP called a stateless protocol?

HTTP is a stateless protocol because the server does not keep any data(state) between two requests.

3. Describe the three phases of establishing a session using client-server protocols such as HTTP?

The three phases of establishing a session using client-server such as HTTP are as follows:

  • The client establishes a TCP connection(or the appropriate connection if the transport layer is not TCP)
  • The client sends in request and waits for a response/answer
  • The server processes the request, sending back the answer along with a status code and appropriate data
